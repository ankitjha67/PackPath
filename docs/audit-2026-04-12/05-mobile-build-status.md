# 05 — Mobile build status

## 🔴 Mobile cannot build. Confirmed.

Inventory of `apps/mobile/android/` and `apps/mobile/ios/`:

```
apps/mobile/android/
├── app/
│   └── src/main/AndroidManifest.xml      ← only this
├── local.properties

apps/mobile/ios/
├── Flutter/
│   ├── Generated.xcconfig                 ← stale, generated by pub get
│   └── flutter_export_environment.sh
└── Runner/
    ├── GeneratedPluginRegistrant.h        ← stale
    ├── GeneratedPluginRegistrant.m
    └── Info.plist                          ← only this is hand-written
```

## What's missing on Android

- `android/build.gradle` (project-level)
- `android/settings.gradle`
- `android/gradle.properties`
- `android/gradle/wrapper/gradle-wrapper.jar` + `gradle-wrapper.properties`
- `android/gradlew` + `android/gradlew.bat`
- `android/app/build.gradle`
- `android/app/src/main/kotlin/.../MainActivity.kt`
- App icons under `android/app/src/main/res/mipmap-*`
- Launch theme XML resources

## What's missing on iOS

- `ios/Runner.xcodeproj/`
- `ios/Runner.xcworkspace/`
- `ios/Runner/AppDelegate.swift` (or .m + .h)
- `ios/Runner/Assets.xcassets/`
- `ios/Runner/Base.lproj/Main.storyboard`
- `ios/Runner/Base.lproj/LaunchScreen.storyboard`
- `ios/Podfile` + `ios/Podfile.lock`
- `ios/Flutter/AppFrameworkInfo.plist`
- `ios/Flutter/Debug.xcconfig` / `Release.xcconfig`

## Net effect

`flutter build apk --debug` cannot succeed. `flutter build ios` cannot succeed. Even `flutter run` on a real device would fail at the platform-build step. Only `flutter analyze` and `flutter test` work because they don't touch the platform projects.

## Why this happened

Earlier sessions wrote `AndroidManifest.xml` and `Info.plist` directly without ever running `flutter create .` to scaffold the surrounding native projects. This is a known gap — `docs/ROADMAP.md` polish-week section explicitly lists:

> - [ ] `flutter create .` to inject the gradle/Xcode wrapping projects

It just was never done.

## How to fix

```bash
cd apps/mobile
# Will create the missing files without overwriting AndroidManifest.xml or Info.plist
flutter create --platforms=android,ios .
flutter pub get
flutter build apk --debug
```

The `flutter create .` command is designed to be re-run safely on an existing project — it only creates files that don't exist. The hand-written `AndroidManifest.xml` and `Info.plist` survive.

After running this, the next gap will be the package name / bundle identifier. The default is `com.example.packpath` — needs to become something like `app.packpath.mobile` (matching the `userAgentPackageName: 'app.packpath.mobile'` already used in `trip_map_screen.dart`).

## Severity

This is a **release blocker for v1**. No design polish or screen restyle matters until this is fixed because nothing can be installed on a real device or submitted to either store.

It's a tractable fix — one command + a package-name rename. Should be Task 0 of the next session, before Task 2 (design system extraction).
